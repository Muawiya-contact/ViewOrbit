### Architecture Summary

#### Firestore Schema (7 collections)
- `users`, `channels`, `videos`, `tasks`, `taskAssignments`, `payoutRequests`, `settings`
- Designed for scale: task definitions separate from assignments; per-user collections for high-volume queries
- Atomic transactions for consistency: reward claims, payout refunds, wallet deductions

#### API Endpoints (20+ routes)
- **User APIs**: Task assignment/progress, payout requests, wallet/points
- **Admin APIs**: Task CRUD, payout approval, conversion-rate settings, user management
- All routes protected by centralized auth middleware (`lib/server/api-auth.ts`)
- Consistent error handling and status codes

#### Service Layer
- `task-assignment-service.ts` – task assignment + creation
- `points-service.ts` – conversion-rate + calculation (server-side only)
- `payout-service.ts` – withdraw/approve/reject with automatic refunds

#### Security & Production Safeguards
✅ Firebase ID token verification on all user endpoints  
✅ Admin session cookie + verification on all admin routes  
✅ Server-side point-to-PKR conversion (never client-provided)  
✅ Atomic transactions prevent race conditions  
✅ Review audit trails (adminId, timestamps)  
✅ Automatic notifications on approvals/rejections

### Firestore Schema at a Glance

```
users/{uid}
├─ points: number
├─ earnedPoints: number
├─ completedVideos: string[]
└─ linkedMethods: { jazzcash?, easypaisa? }

tasks/{taskId}                      [Admin-managed definitions]
├─ title, rewardPoints, isActive
└─ createdBy: adminId

taskAssignments/{assignmentId}      [Per-user instances, high-volume query]
├─ assignedUserId, videoId
├─ status: assigned|in_progress|pending_review|completed
├─ watchProgress, rewardGranted
└─ timestamps: createdAt, acceptedAt, completedAt

payoutRequests/{payoutId}
├─ userId, amountPoints, amountPkr
├─ status: pending|approved|rejected
└─ reviewedBy, reviewedAt

settings/default
├─ pointsPerUnit: 1000
└─ pkrPerUnit: 100
```

### Flow Diagrams

**Task Assignment**  
User clicks assign → GET /api/tasks → Firebase token verified → ensureUserProfile → assignTaskForUser → taskAssignment doc created → Task displayed with video player

**Task Completion**  
All requirements met → POST /api/tasks/complete → Transaction: user.points+10, assignment.status=completed, video.completedViews+1 → Reward claimed

**Payout**  
User requests → POST /api/payouts → Server calculates PKR from settings → Points deducted, payoutRequest created (pending) → Admin reviews → Approve/reject → User notified
